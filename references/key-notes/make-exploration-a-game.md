# Make Exploration a Game

Key note extraída de material sobre exploração em RPG. Framework aplicável para worldbuilding, design de dungeons e campanhas. O argumento central: **exploração é chata quando não há gameplay**.

---

## Ideia central

> **Exploração é chata porque muitos GMs não a tratam como um jogo.**

Combate é divertido porque tem:

- regras
- decisões
- risco
- recompensa

Exploração muitas vezes vira apenas:

```
GM descreve → jogadores escutam → seguem em frente
```

Ou seja, **não existe gameplay**.

A solução proposta:

```
transforme exploração em mini-jogos
```

Quando os jogadores **tomam decisões significativas**, exploração se torna interessante.

---

## O loop correto de exploração

Exploração deve seguir um loop parecido com jogos de puzzle ou investigação:

```
OBSERVAR
    ↓
INVESTIGAR
    ↓
DECIDIR
    ↓
DESCOBRIR
```

Esse loop mantém os jogadores engajados.

---

## 1 — Use "objetos interativos"

Em cada cena ou sala, destaque **poucas coisas interessantes**.

Regra prática:

```
2–4 objetos interativos por área
```

Exemplos:

- uma mesa com gavetas
- um altar estranho
- uma estátua quebrada
- um livro antigo

Isso dá **pontos de decisão** aos jogadores.

---

## 2 — Descreva pouco (para provocar perguntas)

Um erro comum:

```
GM explica tudo
```

Isso elimina o jogo.

Melhor abordagem:

```
descreva só o suficiente
```

| Ruim | Bom |
|------|-----|
| "A mesa tem um compartimento secreto com um mapa." | "A mesa tem duas gavetas." |

Agora os jogadores decidem: abrir esquerda, abrir direita, investigar, quebrar. Isso cria **interação ativa**.

---

## 3 — Evite "uma única opção"

Se há apenas um elemento na cena:

```
exploração vira trilho
```

| Ruim | Melhor |
|------|--------|
| "Há uma estrada. Vocês seguem?" | estrada + trilha na floresta + ruínas próximas |

Agora existe **escolha real**.

---

## 4 — Use objetos aninhados

Objetos podem revelar novos objetos.

```
mesa
 └ gaveta
     └ bolsa
         └ carta secreta
```

Isso cria **exploração em camadas**. Os jogadores sentem que estão **descobrindo coisas**, não apenas recebendo informação.

---

## 5 — Transforme tempo em recurso

Exploração fica interessante quando **tempo importa**.

Exemplo sugerido:

- a cada **10 minutos** os jogadores escolhem ações
- a cada **hora** algo acontece

Eventos possíveis:

- monstros aparecem
- ritual avança
- inimigos se aproximam
- loot desaparece

Assim:

```
explorar custa tempo
```

Logo os jogadores precisam **priorizar**.

---

## 6 — Deixe claro quando não há nada

Outro problema comum: jogadores investigam infinitamente.

Solução:

> "Depois de alguns minutos procurando, vocês não encontram nada."

Isso evita rolagens inúteis e mantém o ritmo.

---

## O modelo prático

Cada área deve conter:

```
1 espaço
+ 2–4 interações
+ possíveis segredos
```

Exemplo — sala de necromante:

- mesa com livros
- círculo mágico no chão
- estátua quebrada

Cada um gera **jogo**.

---

## Template para dungeon design

```
AREA
descrição curta

OBJETOS
1
2
3

SEGREDO
opcional
```

Exemplo:

```
CRIPTA ANTIGA

pilares rachados
sarcófago aberto
altar com runas

segredo: passagem sob o altar
```

Agora os jogadores têm **coisas para fazer**.

---

## Insight mais profundo

Exploração precisa de **agência do jogador**.

| Não é exploração | Exploração real |
|------------------|-----------------|
| GM conta história | jogadores fazem perguntas |

---

## Conexão com videogames

Esse modelo é parecido com jogos de exploração como **Myst**, onde o jogador avança ao **examinar objetos e resolver puzzles ambientais**.

```
exploração = puzzle ambiental
```
